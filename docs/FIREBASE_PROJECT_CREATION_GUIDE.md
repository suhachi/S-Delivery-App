# 🔥 Firebase 프로젝트 생성 및 설정 가이드

이 문서는 개발자가 **Firebase Console**에서 수행해야 할 작업의 상세 순서를 설명합니다.
모든 신규 상점 구축 시 이 절차를 **토씨 하나 틀리지 않고** 따라야 합니다.

---

## 1. 프로젝트 생성 (Create Project)

1.  **[Firebase Console](https://console.firebase.google.com/)** 접속.
2.  **"프로젝트 추가 (+)"** 버튼 클릭.
3.  **프로젝트 이름 입력**:
    - 형식: `[영문상호]-app` (예: `rye-oak-app`)
    - *Tip*: 프로젝트 ID가 이름과 동일하게 생성되는지 확인하세요. (자동으로 난수가 붙으면 '수정' 눌러서 깔끔하게 만드세요)
4.  **Google Analytics**:
    - "이 프로젝트에서 Google 애널리틱스 사용" -> **해제(OFF)** 권장. (초기 단계 불필요, 복잡도 증가 방지)
5.  **"프로젝트 만들기"** 클릭 -> 잠시 대기 -> **"계속"** 클릭.

---

## 2. 요금제 업그레이드 (Blaze Plan)

Cloud Functions(백엔드 로직)를 사용하려면 **무료(Spark) 요금제로는 불가능**합니다.

1.  좌측 메뉴 하단 **"업그레이드"** 또는 **"Spark 요금제"** 태그 클릭.
2.  **"Blaze (종량제)"** 선택.
3.  **청구 계정 선택**:
    - 회사 법인 카드가 연결된 청구 계정을 선택합니다.
    - 예산 알림 설정(월 $10~$20)을 권장합니다.
4.  **"구매"** 클릭.

---

## 3. 서비스 활성화 (Services Init)

왼쪽 "빌드(Build)" 메뉴에서 아래 서비스들을 차례로 활성화합니다.

### 3-1. Authentication (인증)
1.  **Authentication** 클릭 -> **"시작하기"**.
2.  **"Sign-in method"** 탭 -> **"이메일/비밀번호"** 클릭.
3.  **"사용 설정(Enable)"** 스위치 ON -> **"저장"**.

### 3-2. Cloud Firestore (데이터베이스)
1.  **Firestore Database** 클릭 -> **"데이터베이스 만들기"**.
2.  **위치(Region) 선택**:
    - 한국 서비스라면 **`asia-northeast3` (Seoul)** 선택. (매우 중요: 속도 차이 큼)
3.  **보안 규칙(Security Rules)**:
    - **"프로덕션 모드에서 시작"** 선택 -> **"만들기"**.
    - *나중에 코드로 배포할 것이므로 일단 기본값으로 둡니다.*

### 3-3. Storage (이미지 저장소)
1.  **Storage** 클릭 -> **"시작하기"**.
2.  **프로덕션 모드** 선택 -> **"다음"**.
3.  **Cloud Storage 위치**:
    - Firestore와 동일한 리전(`asia-northeast3`) 권장. (설정 불가 시 기본값 `us-central1`도 무방하지만 속도 차이 있음)
4.  **"완료"**.

---

## 4. 웹 앱 등록 및 Config 확보

프론트엔드 코드(`src/lib/firebase.ts`)가 이 프로젝트를 알아볼 수 있게 등록합니다.

1.  **"프로젝트 개요(Project Overview)"** (집 아이콘) 클릭.
2.  상단에 **iOS / Android / Web(</>)** 아이콘 중 **Web(</>)** 클릭.
3.  **앱 닉네임**: 프로젝트 이름과 동일하게 입력 (`rye-oak-app`).
4.  "Firebase 호스팅 설정" 체크박스는 **해제** (CLI로 할 것이므로).
5.  **"앱 등록"** 클릭.
6.  **`const firebaseConfig = { ... }`** 코드가 나옵니다.
    - 이 안에 있는 값들(`apiKey`, `authDomain`, `projectId` 등)을 복사해둡니다.
    - **Step 2-3 (.env 작성)** 단계에서 사용합니다.
7.  **"콘솔로 이동"** 클릭.

---

## ✅ 완료 확인

이제 프로젝트가 준비되었습니다. 터미널로 돌아가서 `firebase login` 및 `firebase init` (또는 SOP의 배포 절차)를 진행할 수 있습니다.
