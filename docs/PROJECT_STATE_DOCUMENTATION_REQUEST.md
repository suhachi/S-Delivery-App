# 📦 KS 심플배달앱 복제 프로젝트 — 상태 문서화 & 자료 요청서

> **목적**: 현재 심플배달앱의 로컬/배포/백엔드 상태를 **100% 문서화**하여, 시간이 지나도 동일한 품질로 **다른 클라이언트에게 복제·제공**할 수 있는 기준 자료를 만든다.
>
> 이 문서는 **AI가 프로젝트를 완전히 이해하기 위해 필요한 모든 자료 요청 목록**이며, 업로드되는 파일들은 `프로젝트 파일추가` 영역에 저장되어 **영구 레퍼런스**로 사용된다.

---

## 0. 🚩 프로젝트 방향 선언 (Critical)

이 프로젝트는 이제 다음 목적을 가진다:

*   ❌ **단일 매장 전용 앱 개발 프로젝트 아님**
*   ✅ **우리가 개발한 심플배달앱을 표준 템플릿으로 유지**
*   ✅ 요청이 들어오면 **복제(Clone) → 세팅 → 배포 → 전달**하는 공급형 프로젝트
*   ✅ 기능 추가는 "템플릿 진화"로만 관리, **개별 클라이언트별 임의 수정 금지**

따라서 모든 상태는 **"지금 이 순간의 기준 상태"를 문서로 고정**해야 한다.

---

## 1. 🧠 AI가 100% 이해해야 할 범위 정의

AI가 이 프로젝트를 "완전히 이해"했다는 것은 아래 5가지를 모두 설명할 수 있는 상태를 의미한다.

1.  **소스코드 구조**: 어떤 폴더/파일이 무엇을 담당하는지
2.  **환경 분기 구조**: 로컬 vs 배포 vs Firebase 연결 방식
3.  **백엔드 동작**: Firestore 컬렉션 구조, 보안 규칙, 인증 흐름
4.  **배포 파이프라인**: 어떤 명령을 언제 실행하는지
5.  **복제 시 변경 포인트**: 클라이언트별로 반드시 바뀌는 것 / 절대 바꾸면 안 되는 것

이를 위해 아래 자료들이 필수적으로 요구된다.

---

## 2. 📂 필수 업로드 요청 자료 (Priority: ★★★)

아래 파일들은 **반드시 업로드**되어야 한다. (이 파일들이 없으면 프로젝트 구조 파악 불가)

### 2-1. 프로젝트 메타 & 설정 파일
> **분석 목적**: 빌드 방식, 배포 대상, 환경변수 구조 파악

*   [ ] `package.json`
*   [ ] `package-lock.json` (또는 `pnpm-lock.yaml`)
*   [ ] `vite.config.ts` (또는 `vite.config.js`)
*   [ ] `tsconfig.json`
*   [ ] `.firebaserc`
*   [ ] `firebase.json`
*   [ ] `.env.template`

### 2-2. Firebase 백엔드 정의 (Most Important)
> **분석 목적**: 보안 모델과 데이터 구조를 고정 문서화

*   [ ] `firestore.rules`
*   [ ] `firestore.indexes.json`
*   [ ] `storage.rules`
*   [ ] **Cloud Functions (서버리스 백엔드 로직)**:
    *   [ ] `functions/src/index.ts` (또는 `index.js`)
    *   [ ] `functions/package.json`
    *   [ ] `functions/tsconfig.json`
*   [ ] **Firebase Console 정보** (스크린샷 또는 텍스트 정리):
    *   Authentication 사용 방식 (Email/Password 활성화 여부 등)
    *   Firestore 컬렉션 구조 요약
    *   Hosting 설정 내역

### 2-3. 프론트엔드 전체 구조
> **분석 목적**: 페이지 흐름, 관리자/고객 분리 매커니즘, 전역 상태 관리 파악

**방법 A (권장)**: `src/` 폴더 전체 압축 업로드
**방법 B**: 주요 파일 개별 업로드 (단, 아래 항목은 **필수** 포함)
*   [ ] `src/main.tsx`
*   [ ] `src/App.tsx`
*   [ ] **Core Logic Files**:
    *   [ ] `src/contexts/*.tsx` (전체: Store, Auth, Cart Context)
    *   [ ] `src/services/*.ts` (전체: API 호출 로직)
    *   [ ] `src/types/*.ts` (전체: 데이터 모델 정의)
*   [ ] `src/pages/**` (주요 페이지 컴포넌트)
*   [ ] `src/components/**` (핵심 공통 컴포넌트)
