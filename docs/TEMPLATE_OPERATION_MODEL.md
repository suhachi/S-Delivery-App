# 단일 배달앱 템플릿 운영 모델

## 1. 개요

이 프로젝트는 **단일 상점 배달앱 템플릿**입니다.  
각 상점(클라이언트)마다 별도의 Firebase 프로젝트와 도메인, 나이스페이 키를 사용하며,  
**코드/아키텍처는 공통 템플릿**으로 유지합니다.

## 2. 개발사 정보

- 개발사: **KS컴퍼니**
- 대표: **석경선, 배종수 (공동대표)**
- 역할:
  - 템플릿 설계/개발/유지보수
  - 상점별 Firebase/Hosting/Functions 운영
  - 상점별 초기 셋업(가게정보+메뉴) 및 결제 연동 지원

## 3. 역할 분담

| 구분 | 개발사 (KS컴퍼니) | 점주(클라이언트) |
|------|-------------------|------------------|
| Firebase 프로젝트 생성 | ✅ | ❌ |
| 앱 기본 코드/인프라 유지보수 | ✅ | ❌ |
| 가게 정보/메뉴 초기 입력 | ✅ (점주 제공 데이터 기반) | 정보 제공 |
| 공지/이벤트 작성 | ⭕ (초기 예시만) | ✅ (운영) |
| 나이스페이 계약 | ❌ | ✅ |
| 나이스페이 키 전달 | ❌ | ✅ (개발사에 전달) |
| 나이스페이 키/환경변수 설정 | ✅ | ❌ |
| 도메인 연결 | ✅ | 도메인 명 의사결정 |
| 일상적인 앱 운영 | ❌ | ✅ (관리자 페이지 사용) |

## 4. 클라이언트별 커스터마이징 항목

- Firebase 프로젝트 ID / API Key
- `stores/{storeId}` 문서 (상점 기본 정보)
- `stores/{storeId}/menus` (메뉴/옵션/가격)
- NICEPAY:
  - `.env` → `VITE_NICEPAY_CLIENT_ID`
  - Functions config → `nicepay.secret_key`
- Firebase Hosting 커스텀 도메인

## 5. 복제 플로우 개요

1. 템플릿 리포지토리 복제
2. 새 Firebase 프로젝트 생성 및 `.env` 설정
3. 점주로부터 가게정보/메뉴/이미지 수집
4. Firestore에 상점/메뉴 데이터 입력
5. 나이스페이 계약 후 키 수령
6. ENV/Functions config에 키 설정
7. 커스텀 도메인 연결
8. 최종 QA 후 점주에게 인계
