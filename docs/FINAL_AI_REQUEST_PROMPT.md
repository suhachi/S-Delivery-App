# 📌 Anti-Gravity 최종 파일 요청 프롬프트 (한 번에 끝내는 버전)

## 🎯 Role

너는 **KS 심플배달앱(simple-delivery-app)**의 실제 소스코드와 운영 구조를 알고 있는 개발 에이전트다.
이 프로젝트는 이제 단일 매장 앱이 아니라,

> **심플배달앱을 복제(Clone)하여 여러 클라이언트(상점)에 제공하는 마스터 템플릿 프로젝트**

로 전환되었다.

현재 나는 이미 대부분의 구조·코드·배포 흐름을 문서로 확보했으며,
**아래 3개 파일만 확보되면 프로젝트 상태를 100% 확정하고 문서화를 완료할 수 있다.**

---

## 🧩 Task (중요)

아래 **3개 파일의 “전문(Full Text)”만 제공하라.**
추가 설명, 요약, 발췌는 필요 없다.

조건:

* ❌ 코드 일부 발췌 금지
* ❌ 요약/설명 금지
* ❌ 추정/재작성 금지
* ⭕ **실제 프로젝트에 존재하는 원본 내용 그대로 제공**

---

## 🔴 최종 필수 요청 파일 (정확히 이 3개만)

### 1️⃣ `storage.rules` (전문)

목적:

* Firebase Storage 접근 제어 모델 확정
* 관리자/점주/고객 권한 범위 고정
* 복제 시 보안 사고 여부 판단

요청:

```
storage.rules  (파일 전체 전문)
```

---

### 2️⃣ `package.json` (root, 전문)

목적:

* 빌드/배포 스크립트
* Node / Firebase / Vite / React 의존성 확정
* 복제 시 환경 불일치 방지

요청:

```
package.json  (루트, 파일 전체 전문)
```

---

### 3️⃣ 관리자 초기 세팅 기준 문서 (택 1)

목적:

* 점주가 앱에서 "어디서부터 시작하는지"를 공식적으로 고정
* 클라이언트 전달 문구 및 SOP 자동화

아래 중 **하나만 제공하면 충분**:

* `README.md` (실제 운영 기준)
* 또는 관리자 초기 플로우 설명 MD

  * 예: `ADMIN_SETUP_FLOW.md`
  * 내용 필수 요소:

    * 관리자 로그인
    * 상점 생성
    * 메뉴 등록

---

## 📦 Output 형식

* 각 파일은 **개별 파일 그대로** 제공하거나
* 여러 파일일 경우 **ZIP 1개**로 묶어 제공
* 파일명은 반드시 실제 프로젝트와 동일해야 한다.

---

## 🚫 금지 사항

* 이미 제공된 `.firebaserc`, `firebase.json`, `firestore.rules`, `.env.template`, `check-deploy.mjs`를 다시 설명하지 말 것
* "이 파일은 이런 역할입니다" 같은 설명으로 대체하지 말 것

---

## ✅ 완료 기준

아래 조건이 충족되면 이 요청은 완료다:

> 위 3개 파일이 원본 전문으로 제공됨

이후에는 **추가 파일 요청 없이**,

* 현재 앱 상태 100% 확정 문서
* 심플배달앱 복제 불변 규칙
* 클라이언트용 / 내부용 SOP 분리본

을 바로 생성할 수 있다.

---

이 요청은 **시간 낭비를 막기 위한 최종 요청**이다.
정확하게, 빠짐없이 제공하라.
