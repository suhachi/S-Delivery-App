# KS 단일 배달앱 - 클라이언트 온보딩 체크리스트

이 문서는 템플릿을 복제하여 새 상점을 오픈할 때 확인해야 할 필수 항목을 정리합니다.

## 1. 프로젝트 및 인프라 (KS 담당)

- [ ] **Git 리포지토리 복제**
  - [ ] 템플릿(`simple-delivery-app`) → 새 리포지토리(`simple-delivery-shop-[이름]`) 생성
- [ ] **Firebase 프로젝트 연결**
  - [ ] 새 Firebase 프로젝트 생성 (콘솔)
  - [ ] `.firebaserc` : `default` 프로젝트 ID 수정
  - [ ] `.env` : Firebase Web Config 값 교체 (`VITE_FIREBASE_API_KEY` 등)
  - [ ] Firestore, Auth, Storage, Functions 활성화
- [ ] **보안 및 환경변수 설정**
  - [ ] `firestore.rules` 배포 (템플릿 원본 유지)
  - [ ] `storage.rules` 배포
  - [ ] 나이스페이 키 설정 (테스트 또는 실서비스)
    - [ ] `.env`: `VITE_NICEPAY_CLIENT_ID`
    - [ ] `functions:config`: `nicepay.secret_key`
- [ ] **초기 데이터 정리**
  - [ ] `stores`, `orders`, `reviews` 컬렉션 초기화 확인 (비어있는 상태)
  - [ ] 테스터/임시 계정 삭제

## 2. 상점 운영 설정 (점주/운영자 담당)

### 2.1 상점 기초 정보
- [ ] **최초 관리자 로그인 및 상점 생성**
  - [ ] `/admin` 접속 → 로그인
  - [ ] "상점 생성 마법사" 진입 확인
- [ ] **필수 정보 입력**
  - [ ] 상호명, 전화번호, 주소
  - [ ] 영업시간 (오픈/마감) 및 휴무일
  - [ ] 최소 주문 금액, 기본 배달팁
  - [ ] 로고 이미지 업로드

### 2.2 메뉴 구성
- [ ] **카테고리 등록** (예: 메인, 사이드, 음료)
- [ ] **메뉴 등록**
  - [ ] 메뉴명, 가격, 설명
  - [ ] 메뉴 이미지 업로드 (권장: 1:1 비율)
  - [ ] **옵션 그룹 설정** (선택 사항)
    - [ ] 필수/선택 여부, 최대 선택 개수
    - [ ] 옵션 항목 및 추가격

### 2.3 부가 기능 활성화
- [ ] **전화번호 연동 확인**
  - [ ] 회원가입/주문 시 전화번호 입력 필수 동작 확인
- [ ] **공지사항 등록** (오픈 이벤트 등)
- [ ] **쿠폰 발행** (옵션)
  - [ ] 회원가입 축하 쿠폰, 할인 쿠폰 등 생성

## 3. 결제 및 주문 테스트 (최종 검증)

- [ ] **결제 연동 확인**
  - [ ] 실제 결제 시도 (100원 또는 테스트 모드)
  - [ ] 결제 승인 완료 및 주문 생성 확인
- [ ] **주문 접수 프로세스**
  - [ ] 앱: 주문 완료 화면 진입
  - [ ] 관리자: 주문 접수 알림(사운드) 및 목록 갱신 확인
  - [ ] 영수증 출력 팝업(또는 자동) 동작 확인
- [ ] **취소/환불 테스트**
  - [ ] 관리자에서 '주문 취소' 시 결제 취소 연동 여부
