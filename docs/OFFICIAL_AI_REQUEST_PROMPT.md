# 🤖 Anti-Gravity 요청용 공식 프롬프트

아래 프롬프트를 **그대로 복사해서 Anti-Gravity(또는 Cursor/Opal 등 자동화 개발 에이전트)**에게 전달하세요.
이 프롬프트의 목적은 **현재 심플배달앱 프로젝트를 100% 이해·복제 가능한 상태로 만들기 위해 필요한 모든 원본 파일을 정확한 형식으로 제공받는 것**입니다.

---

## 📌 Role

너는 **KS 심플배달앱(simple-delivery-app) 프로젝트의 실제 소스코드와 배포 상태를 가장 잘 알고 있는 개발 에이전트**다.
이 프로젝트는 이제 단일 매장 앱이 아니라,
**"심플배달앱을 복제하여 여러 클라이언트(상점)에게 제공하는 마스터 템플릿 프로젝트"**로 전환되었다.

따라서 나는 현재 프로젝트의 **로컬 상태 / 배포 상태 / Firebase 백엔드 상태를 100% 정확히 문서화**해야 하며,
이를 위해 아래 파일들을 **"원본 그대로"** 제공받아야 한다.

---

## 🎯 Task (중요)

아래 **🔴 필수 파일들**을 반드시 제공하라.

조건:

* 코드 일부 발췌 ❌
* 설명용 스니펫 ❌
* 문서 요약 ❌
* **실제 프로젝트에 존재하는 원본 파일만 허용 ⭕**

제공 방식은 **파일 단위 그대로**이거나,
여러 개일 경우 **폴더 구조를 유지한 ZIP 파일**로 제공하라.

---

## 🔴 필수 제공 파일 목록 (절대 누락 금지)

### 1️⃣ Firebase 프로젝트 연결 핵심 파일 (최우선)

> 이 파일이 없으면 로컬 ↔ 배포 ↔ 클라이언트 복제 흐름을 이해할 수 없다.

반드시 제공:

```
.firebaserc
firebase.json
```

---

### 2️⃣ Firebase 보안 규칙 원본

> 점주 / 관리자 / 고객 권한 모델과 데이터 분리 구조를 파악하기 위함

반드시 제공:

```
firestore.rules
storage.rules
firestore.indexes.json (존재한다면 반드시 포함)
```

---

### 3️⃣ 환경변수 템플릿 (실결제 사고 방지 목적)

> 어떤 값이 클라이언트별로 바뀌고, 어떤 값은 절대 건드리면 안 되는지 판단하기 위함

반드시 제공:

```
.env.template
```

⚠️ 주의:

* `.env` 실파일을 주지 말 것
* 민감 키가 있다면 반드시 비워둔 상태로 제공할 것

---

### 4️⃣ 배포 스크립트 & 안전장치 전체

> 템플릿/클라이언트 프로젝트에 잘못 배포되는 사고를 막기 위한 구조 파악 목적

반드시 제공:

```
package.json (root)

scripts/
 └ check-deploy.mjs (전체 원본)
```

---

### 5️⃣ 관리자 초기 세팅 기준 문서 (택 1 이상)

> 점주가 앱에서 "어디서부터 시작하는지"를 명확히 정의하기 위함

아래 중 **최소 1개 이상 제공**:

* `README.md` (실제 운영 기준)
* 또는 관리자 초기 플로우를 설명한 MD 파일

  * 예: `ADMIN_SETUP_FLOW.md`
  * 내용: 관리자 로그인 → 상점 생성 → 메뉴 등록

---

## 🔵 가능하면 함께 제공 (권장)

아래는 필수는 아니지만, 제공 시 프로젝트를 즉시 **복제 비즈니스용 마스터 템플릿**으로 격상할 수 있다.

* 현재 실제 배포 중인 URL 목록 (prod / test)
* 운영 중인 Firebase 프로젝트 ID 목록
* "심플앱 고정 기능 / 옵션 기능" 구분 문서
* 클라이언트 전달용 기본 안내 메일 또는 문구 템플릿
