# 📋 런칭 준비 분담표 (R&R Matrix)

새로운 배달앱을 런칭하기 위해 각 주체(클라이언트, 개발자, AI)가 준비하고 수행해야 할 역할을 명확히 구분한 문서입니다. `STANDARD_LAUNCH_SOP.md`를 실행하기 전 체크리스트로 활용하세요.

---

## 1. 👤 클라이언트 (점주) 준비사항
**소스코드나 기술적인 내용은 전혀 몰라도 됩니다.** 아래 정보와 자산만 준비해달라고 요청하세요.

### 필수 (런칭 전)
- [ ] **상호명 (앱 이름)**: 예) `라이옥 쌀국수`
- [ ] **관리자 계정용 이메일**: 예) `owner@ryeoak.com` (실제 수신 가능한 메일 권장)
- [ ] **사업자 등록증**: (전자상거래업 신고용)
- [ ] **로고 이미지**: 앱 상단 및 아이콘으로 쓸 이미지 (512x512px, PNG 권장)

### 필수 (런칭 후 ~ 오픈 전)
- [ ] **메뉴 데이터**: 메뉴명, 가격, 설명, 음식 사진
- [ ] **NICEPAY 가맹 계약**: 결제 대행사 계약 진행
- [ ] **도메인 (선택)**: 별도 도메인(`ryeoak.com`)을 쓰고 싶다면 도메인 구입 필요 (없으면 Firebase 기본 도메인 사용)

---

## 2. 🧑‍💻 개발자 (User/팀장님) 준비사항
**실무적인 인프라 세팅과 배포, 전달**을 담당합니다.

### 인프라 & 환경 (Infrastructure)
- [ ] **Firebase 프로젝트 생성**: 콘솔에서 새 프로젝트(`rye-oak-app`) 생성 및 **Blaze 요금제** 설정.
- [ ] **서브도메인/도메인 준비**:
    - Firebase Hosting에서 제공하는 기본 도메인(`web.app`)을 쓸지,
    - 회사 보유 도메인의 서브도메인(`ryeoak.ks-delivery.com`)을 연결할지 결정 및 DNS 설정.

### 기술 실행 (Execution)
- [ ] **Git Clone & Setup**: SOP 문서에 따라 리포지토리 복제 및 초기화.
- [ ] **Config 설정**:
    - `.firebaserc`: 프로젝트 ID 연결.
    - `.env`: API Key 등 환경변수 세팅 (PG 키는 공란).
- [ ] **배포 (Deployment)**:
    - 터미널에서 `npm run build` 및 `npm run firebase:deploy...` 명령어 실행.
    - 배포 전 스크립트(`check-deploy.mjs`) 통과 확인.

### 전달 (Handover)
- [ ] **초기 검수**: 관리자 페이지 접속하여 "상점 설정 마법사" 뜨는지 확인.
- [ ] **계정 전달**: 점주에게 관리자 URL과 초기 로그인 정보 전송.

---

## 3. 🤖 AI (Agent/System) 준비사항
이미 준비되어 있거나, 개발자가 요청하면 즉시 수행할 수 있는 영역입니다.

### 기 제공된 자산 (Assets)
- [x] **Master Template (v1.0.0)**: 완벽하게 동작하는 불변의 소스코드 원본.
- [x] **Safety Script (`check-deploy.mjs`)**: 배포 사고를 막아주는 자동 검증 도구.
- [x] **SOP (`STANDARD_LAUNCH_SOP.md`)**: 누구나 따라 할 수 있는 표준 운영 절차서.

### 기술 지원 (Assistance)
- [ ] **에러 트러블슈팅**: 배포 중 발생하는 예상치 못한 빌드/배포 에러 분석 및 해결.
- [ ] **맞춤형 수정**: 템플릿 범위를 벗어나는 특수한 요구사항(커스텀 기능)이 있을 경우 코드 수정 지원.
- [ ] **데이터 마이그레이션**: 추후 대규모 데이터 처리가 필요할 때 스크립트 작성.

---

## 📝 런칭 프로세스 요약

1.  **클라이언트**가 정보(상호, 로고 등)를 **개발자**에게 전달.
2.  **개발자**는 Firebase 프로젝트와 도메인을 준비하고, **AI**가 만든 템플릿을 복제(Clone).
3.  **개발자**가 SOP에 따라 설정 및 배포 수행 (필요 시 **AI**가 에러 해결).
4.  **개발자**가 결과물(URL)을 **클라이언트**에게 전달.
5.  **클라이언트**가 메뉴 등록 및 PG 계약 후 오픈.
