# 🧪 v2.0 Local Verification Checklist (로컬 테스트 가이드)

`npm run dev` 실행 후, 다음 항목들을 순서대로 확인해 주세요.

## 📊 1. 관리자 매출 통계 (Admin Stats)
*   [ ] **접속**: 관리자 사이드바 > `매출 통계` 클릭.
*   [ ] **데이터 확인**: 상단 카드에 "기간 내 총 매출"과 "주문 수"가 `0`이 아닌 숫자로 나오는지 확인. (기존 주문이 있다면)
*   [ ] **차트 확인**:
    *   `일별 매출 추이` 막대 그래프가 표시되는가?
    *   `주문 건수 추이` 꺾은선 그래프가 표시되는가?
*   [ ] **기간 변경**: `최근 7일` / `최근 30일` 버튼 클릭 시 숫자가 변경되는가?

## 🔍 2. 회원 검색 (Member Search)
*   [ ] **접속**: 관리자 사이드바 > `회원 관리` 클릭.
*   [ ] **목록 로딩**: 가입된 회원 목록이 테이블에 표시되는가?
*   [ ] **검색 테스트**:
    *   검색창에 **본인 이름** 입력 -> 해당 회원만 필터링되는지 확인.
    *   검색창에 **전화번호 뒷자리** 입력 -> 검색되는지 확인.

## 🛵 3. 배달 대행 설정 (Delivery Settings)
*   [ ] **접속**: 관리자 사이드바 > `상점 설정` 클릭.
*   [ ] **스크롤**: 페이지 하단 `배달 대행 연동` 카드로 이동.
*   [ ] **UI 확인**:
    *   기본값인 `연동 안 함`이 선택되어 있는가?
    *   셀렉트 박스를 `바로고 (Barogo)`로 변경 시, 하단에 API Key 입력창이 나타나는가?
    *   `Webhook URL`이 올바르게 표시되는가? (예: `https://us-central1-...`)
*   [ ] **데이터 입력 & 저장**:
    *   가상의 값 입력 (예: ShopID: `test`, keys: `1234`)
    *   `변경사항 저장` 클릭 -> "저장되었습니다" 알림 확인.
    *   새로고침 후에도 `바로고` 설정이 유지되는지 확인.

---

### ✅ 모든 항목이 통과되면 배포 준비 완료!
테스트 중 오류가 발생하면 즉시 알려주세요.
